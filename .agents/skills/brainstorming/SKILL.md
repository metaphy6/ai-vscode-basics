# Brainstorming

## When to use

The request is ambiguous *and* the cost of picking the wrong direction is
high. Diverge before converging.

## Procedure

1. **Restate the goal** in your own words.
2. **List 3–5 candidate approaches**, each in one sentence. Force breadth
   — include at least one "obvious" option and at least one "weird" option.
3. **For each candidate**, name:
   - the main trade-off,
   - the cheapest way to disprove it (a spike, a sketch, a quick read).
4. **Recommend one** and say why. The recommendation is a starting point,
   not a decision — the user can redirect.
5. **Stop.** Don't implement until the user picks.

## Anti-patterns

- ❌ Picking one approach and asking "should I do this?" (that's not brainstorming, that's seeking permission).
- ❌ Listing 12 options nobody will read.
- ❌ Brainstorming when the answer is obvious — just do it.
- ❌ Skipping the "weird" option because you'd be embarrassed to recommend it.
- ❌ Starting to code mid-brainstorm.
